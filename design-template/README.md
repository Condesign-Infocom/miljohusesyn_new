# Miljöhusesyn design template — for AI use

This folder is a self-contained design template for the Miljöhusesyn look & feel. It is a
**reference/source kit** — it is independent of the parent app in `E:\Projekt\Miljohusesyn`
and does not affect it. An AI (or developer) can read from here and copy parts into the app.

There are **two parallel deliverables** sharing the same design tokens and section content:

## 1. Zero-build static HTML template (this folder's root)

Plain HTML sections stitched by a tiny Python script — no npm, no framework.

- **Read [`AGENTS.md`](AGENTS.md) first** — it is the contract: how to add pages/sections,
  the conventions, and the rules (e.g. reuse existing utility classes; nav lives in one place).
- `template.config.json` — machine-readable catalog of sections, pages, nav, design tokens,
  and captioned images.
- Edit files in `src/`, then run `python build.py` → output in `dist/` (already built;
  open `dist/index.html` to preview).

## 2. Svelte component library (`svelte/`)

The same design as reusable **Svelte 5 / SvelteKit** components (Tailwind v4, lucide-svelte,
Fraunces). Best fit if you're working in the SvelteKit app.

- **Read [`svelte/README.md`](svelte/README.md)** — full component + props reference.
- 6 primitives (`Button`, `Card`, `Badge`, `SearchBar`, `NavLink`, `Icon`) and 9 sections
  (`Header`, `Hero`, `PartnersBand`, `FeatureGrid`, `Faktabank`, `Calculations`, `News`,
  `Contact`, `Footer`), all importable from `svelte/src/lib/index.js`.
- Run the demo: `cd svelte && npm install && npm run dev`.
- **Note:** these components are **JavaScript**; the parent app is **TypeScript**. Adapt types
  when integrating (props are documented in `svelte/README.md`).

## Design tokens (shared by both)

Colors `cream #faf6ec`, `cream-2 #f3ecda`, `bark #1e2a22`, `ink #243027`, `line #e3dcc6`,
`leaf #2f5d3a`, `leaf-2 #244a2e`, `sage #88a878`, `sage-2 #c8d6b6`, `mute #5b6a5f`; display
font **Fraunces**. Static template: tokens documented in `src/styles/tokens.css`. Svelte:
declared in `svelte/src/app.css` `@theme`.

## What's NOT included

Build artifacts and dependencies are intentionally omitted (regenerate them):
`svelte/node_modules` (run `npm install`), `svelte/.svelte-kit`, and the static `dist/` was
generated by `python build.py`. Design rationale lives in `docs/superpowers/specs` and
`docs/superpowers/plans`.
